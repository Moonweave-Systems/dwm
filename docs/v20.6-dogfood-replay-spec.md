# V20.6 Dogfood Replay Spec

Status: implemented dogfood replay gate in `scripts/dwm_dogfood_replay.py`.

## Research and Prior Art

Release gates and reviewer gates can still drift if the canonical dogfood chain
is only described in documentation. The useful next step is a replay artifact:
run the known deterministic dogfood commands, capture command output, inspect
the final DWM status, and prove the source repository status did not change.

## Product Position and Non-Goals

V20.6 is an evidence gate after V20.5. It makes the existing dogfood path
replayable as a release check.

Non-goals:

- do not execute live Codex, Claude, OMX, or shell adapters beyond the existing
  local deterministic DWM scripts,
- do not attach sessions, create worktrees, or fan out workers,
- do not publish packages or deploy services,
- do not treat generated `out/` files as source truth.

## Workflow Architecture

The replay runs the canonical dogfood commands from `scripts/dwm.py`, then runs:

- `python scripts/dwm.py status --run out/v9/v32-semantic-dogfood --json`,
- `python scripts/dwm.py next --run out/v9/v32-semantic-dogfood --json`.

The accepted terminal state requires:

- repo status unchanged before and after replay,
- final status `workflow-complete`,
- trusted next summary,
- `recommendation.action` equal to `complete`,
- every command return code equal to zero.

## Execution Model

`scripts/dwm_dogfood_replay.py replay --out out/dogfood-replay/<replay_id>`
writes:

- `.dwm_dogfood_replay-owned.json`,
- `commands.json`,
- `replay.json`,
- `replay.md`,
- `status.json`.

The output directory must be under `out/dogfood-replay/` and must be either new
or already owned by the V20.6 replay sentinel.

## Safety and Verification Gates

The replay blocks rather than accepts when:

- `ERR_DOGFOOD_COMMAND_FAILED`: any replay command fails,
- `ERR_DOGFOOD_REPO_DIFF_CHANGED`: repo status changed during replay,
- `ERR_DOGFOOD_FINAL_STATUS`: final status, trust, or recommendation action is
  not the expected complete state.

## Evaluation Fixtures

`fixtures/v20.6/manifest.json` covers:

- positive: real dogfood replay reaches accepted status,
- negative: synthetic command failure is blocked,
- negative: synthetic repo status drift is blocked,
- negative: synthetic final status mismatch is blocked.

## Release Plan

Add V20.6 to release commands and contract checks after V20.5. This gate
confirms the dogfood path remains replayable; it does not claim live adapter
execution.
